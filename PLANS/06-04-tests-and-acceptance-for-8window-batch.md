# 06-04: 8-Window Batch 테스트 및 수용 기준 세부 계획

## 목적

1. 8-window 배치의 핵심 행위를 테스트로 고정한다.
2. 정책 회귀(단일 진입점/CLI 금지/venv 규칙)를 방지한다.

## 단위 테스트 항목

1. window 8개 생성 커맨드 구성 검증.
2. 1-window 검증 제거 및 8-window 검증 로직 확인.
3. case 디렉토리(`case_01`~`case_08`) 생성 규칙 검증.
4. `wait-all` 종료 조건(8개 `exit.code`) 검증.
5. summary 집계 포맷 검증.

## 모의 통합 테스트 항목

1. 8개 모두 성공 시 `success=True` 및 다운로드 호출 검증.
2. 일부 실패 시 `success=False`지만 다운로드 호출 유지 검증.
3. 다수 실패 시에도 `wait-all` 후 종료 검증.
4. `screen` 생성 실패 시 적절한 exit code 검증.
5. 다운로드 실패 시 실행 실패와 분리 보고 검증.

## 정책 회귀 검증

1. `run_pipeline` 단일 진입점 유지.
2. CLI import/엔트리포인트 미추가 유지.
3. 테스트 실행 명령은 `.venv/bin/python` 기준으로 문서화.

## 수용 기준

1. 8개 모두 종료 감지 성공.
2. 성공/실패 무관 전량 회수 성공.
3. case별 로그/exit code 누락 0건.
4. 요약 문자열에 실패 case 정보 포함.
