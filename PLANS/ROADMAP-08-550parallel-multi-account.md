# ROADMAP 08: 최대 7계정, 목표 550 처리량

## 목표/범위

목표:

1. 다계정(최대 7개)으로 처리량 확장
2. 라이선스 550 한도 내 최고 처리량 달성
3. 계정별 실패/성능 편차를 제어하며 안정 운영

## 아키텍처 변경점

1. 계정 셰어딩 레이어 도입
2. 글로벌 스케줄러(FIFO + 우선순위) 도입
3. 라이선스 토큰 기반 스로틀러 도입

## 인터페이스/API/스키마 변경

1. 계정 레지스트리 구성 추가:
   - `account_id`, `host_alias`, `max_jobs`, `enabled`
2. 글로벌 스케줄링 상태 추가:
   - `assigned_account`, `priority`, `license_tokens_reserved`

## 실행 플로우

1. 글로벌 큐에서 작업 추출
2. 계정 상태/슬롯/라이선스 기반으로 계정 배정
3. 계정별 07 실행기 호출
4. 완료/실패 이벤트를 글로벌 상태에 반영
5. 실시간 처리량 기준으로 분배 가중치 조정

## 리소스 정책(코어/라이선스/계정/잡)

1. 계정 최대 수: 7
2. 글로벌 라이선스 상한: 550
3. 계정별 동시 잡 상한은 운영 설정으로 조정
4. 스로틀러는 라이선스 상한 초과 제출 금지

## 실패 처리 및 복구 절차

1. 계정 장애 격리: 장애 계정 `disabled` 전환
2. 장애 계정에 배정된 작업 재큐잉
3. 라이선스 부족 시 대기열 백오프 적용

## 테스트/수용 기준

1. 계정 셰어딩 정확성 검증
2. 라이선스 스로틀러 상한 준수 검증
3. 장애 계정 격리/복구 검증
4. 목표 처리량 도달 여부 검증

## 운영 체크리스트

1. 계정별 SSH/권한/환경 상태 확인
2. 계정 활성/비활성 정책 확인
3. 라이선스 사용량 모니터링 확인

## 롤백 기준

1. 다계정 불안정 시 07 단계(단일 계정 80 병렬)로 롤백
