# 06-03: 성공/실패 무관 결과 회수 세부 계획

## 목적

1. 케이스 성공/실패와 관계없이 원시 산출물을 전량 회수한다.
2. 실행 실패와 다운로드 실패를 구분해 기록한다.

## 회수 대상

1. 공통 로그:
   - `remote_run.log`
   - `screen` 관련 진단 로그
2. case별 파일:
   - `case_XX/run.log`
   - `case_XX/exit.code`
   - 생성된 결과 파일(`*.aedtresults` 등)
3. 집계 파일:
   - `case_summary.json` 또는 동등 텍스트 요약

## 아카이브 정책

1. `results.tgz`에 공통 로그 + case별 로그/exit code + 결과 파일 포함.
2. `.venv`, 캐시 등 대용량 불필요 항목은 제외.
3. 실패 case 파일도 동일 우선순위로 포함.

## 에러 분리 규칙

1. 원격 실행 실패:
   - case exit code 실패, 스크립트 실행 예외.
2. 다운로드 실패:
   - `scp` 실패, 압축 해제 실패.
3. 두 실패를 summary에서 별도로 표기.

## 데이터 보존 우선 규칙

1. 분석은 후속 단계에서 수행한다.
2. 이번 단계는 “먼저 회수”를 최우선으로 한다.
3. 회수 완료 전 원격 정리 수행 금지.
